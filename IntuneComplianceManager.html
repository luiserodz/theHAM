Intune Compliance Manager

Monitor and manage device compliance and application deployment status

Authentication Required: Click "Sign in with Microsoft" to connect to your Intune environment.

Step 1: Register an Application in Azure AD

  1. Go to the Azure Portal - App Registrations
  2. Click New registration
  3. Configure the application:
    • Name: Intune Compliance Manager
    • Supported account types: Accounts in this organizational directory only
    • Redirect URI: Single-page application (SPA)
  4. Add this redirect URI:
    Important: Copy and add this exact URI to your app registration:
  5. Click Register

Step 2: Configure API Permissions

  1. In your app registration, go to API permissions
  2. Click Add a permission â†’ Microsoft Graph â†’ Delegated permissions
  3. Add these permissions:
    • User.Read - Sign in and read user profile
    • DeviceManagementManagedDevices.Read.All - Read device compliance data
    • DeviceManagementApps.Read.All - Read application data
  4. Click Grant admin consent (requires admin rights)

Step 3: Get Your Application Details

  1. Go to the Overview page of your app registration
  2. Copy the Application (client) ID
  3. Copy your Directory (tenant) ID (optional, for single-tenant apps)

Step 4: Connect to Intune

  1. Click the Sign in with Microsoft button below
  2. Enter your Application ID when prompted
  3. Sign in with your Microsoft 365 admin account
  4. Start managing compliance data!

Required Permissions Summary

Microsoft Graph API Permissions:
  • User.Read - Required for authentication
  • DeviceManagementManagedDevices.Read.All - View device compliance status
  • DeviceManagementApps.Read.All - View application deployment data

Note about Application Deployment Data: Microsoft Graph API has limitations for application deployment statistics. The install summary data may show zeros if:
  • Applications haven't been deployed to any devices yet
  • Additional permissions are required (DeviceManagementApps.ReadWrite.All)
  • The data needs to be accessed through device-specific queries
For detailed deployment data, you may need to use the Intune portal directly or query device install states individually.
